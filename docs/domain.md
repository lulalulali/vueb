# 域名

## 更改域名显示想要的文档的方式

### 1. 路由配置

通过设置路由，可以将不同的 URL 映射到不同的文档或页面。在前端开发中，可以使用路由库（如 React Router、Vue Router 等）来管理 URL 和页面之间的映射关系。

### 2. 后端配置

如果是在服务器端进行开发，可以在后端代码中配置路由，使不同的 URL 请求能够返回不同的文档或页面内容。这通常涉及使用服务器端框架（例如 Express.js、Django、Spring 等）来处理路由和请求。

### 3. 文件系统结构

另一种方式是直接在文件系统中组织文档结构。通过在特定文件夹下创建不同的文件或子文件夹，可以实现不同路径对应不同的文档或页面。

关于如何更改域名显示特定文档的方法，具体取决于你使用的技术栈和开发框架。你可能需要编辑路由配置、后端代码或文件系统结构来实现这个目标。如果你使用特定的框架或工具，可以查阅相关文档或教程，寻找关于路由配置或页面组织的指南。

## url 路径与相对文件路径 特征

URL（Uniform Resource Locator）是用于定位资源的地址。它是在网络上标识和定位资源（比如网页、图像、文件等）的方式。URL 通常由多个部分组成：

- **协议**：通常是 `http://` 或 `https://`，指定了访问资源所使用的协议。
- **域名**：指定了资源所在的主机或服务器的名称，比如 `www.example.com`。
- **路径**：资源在服务器上的位置，指定了服务器上文件的位置或具体的路由，以便访问特定的内容。
- **查询参数**：可选的部分，用于向服务器传递额外的参数，以影响资源的呈现或请求的处理。

相对文件路径是文件系统中用于定位文件相对位置的方式。它描述了一个文件相对于另一个文件或当前工作目录的位置。相对路径可以是：

- **相对于当前文件夹的路径**：比如 `./file.txt`，表示在当前文件夹中查找文件 `file.txt`。
- **相对于父文件夹的路径**：比如 `../parentFolder/file.txt`，表示在当前文件夹的父文件夹中查找 `file.txt`。

### 特征

- **URL 路径特征**：
  - 包含协议、域名、路径等组成部分。
  - 用于在网络上定位和访问资源。
  - 可能包含查询参数来影响请求的行为。

- **相对文件路径特征**：
  - 描述了文件相对于其他文件或当前工作目录的位置。
  - 可以使用相对于当前文件夹或父文件夹的路径表示位置。
  - 在文件系统中使用，用于定位文件或文件夹的位置。

两者的主要区别在于用途和应用场景。URL 路径用于在网络上定位资源，而相对文件路径用于在文件系统中定位文件或文件夹。相对文件路径更侧重于描述文件在文件系统中的位置关系，而 URL 更侧重于在网络上定位资源的位置。
